# Carrousel LinkedIn — Les skills de Claude

**Framework : PAS (Problem-Agitate-Solution)**
*Pourquoi PAS :* L'audience ignore souvent les vraies capacités de Claude (Problème), sous-utilise l'outil (Agitation), et le carrousel révèle le plein potentiel (Solution). Structure idéale pour un contenu éducatif à fort engagement.

---

```markdown
# Vous utilisez Claude comme un *moteur de recherche*.
Il peut faire 10x plus. Voici ses vrais skills — classés, concrets, actionnables.

## [toc] Ce qu'on couvre
1. Rédaction & Copy
2. Code & débogage
3. Analyse & recherche
4. Raisonnement long
5. Agents & automatisation
6. Ce que Claude refuse (et pourquoi c'est bien)

## [pillars] Écriture — bien au-delà du brouillon
### Copy & persuasion
Emails, posts LinkedIn, scripts, landing pages. Il adapte le ton à votre marque, reformule sans perdre la voix.

### Contenu long
Articles, rapports, études de cas. Structurés, sourcés, cohérents sur 5 000+ mots sans perdre le fil.

### Créatif
Scénarios, nouvelles, dialogues. Avec de vraies contraintes narratives respectées.

## [compare] Ce que vous faites vs ce que vous *pourriez* faire
Même outil. Usage complètement différent.

LEFT: Usage basique
- « Résume ce texte »
- « Traduis ça en anglais »
- « Écris un email »

RIGHT: Usage avancé
- « Analyse ce document, identifie les contradictions, donne un plan de contre-argumentation »
- « Code cette fonction en Python avec tests unitaires et gestion d'erreurs »
- « Joue le rôle d'un CFO sceptique et challenge mon business plan »

## [stat] Raisonnement & analyse
200k
tokens de contexte — soit ~500 pages analysées en une seule requête
Claude Opus 4 lit, compare et synthétise des documents entiers. Rapports financiers, contrats, bases de code — sans perdre un détail.

## [flow] Code — du bug au projet complet
Role · Debug · Build · Review — quatre modes, un seul outil.

1. Debug — Identifie l'erreur, explique pourquoi, corrige avec contexte
2. Build — Génère du code fonctionnel avec tests et documentation
3. Review — Audite votre codebase, détecte les failles de sécurité
4. Agents — Exécute des tâches longues en autonomie via Claude Code

## [pillars] Ce que Claude *refuse* — et c'est une feature
### Pas de contenu dangereux
Armes, manipulation, fraude. Les garde-fous sont intentionnels, documentés, et non négociables.

### Honnêteté radicale
Claude dit quand il ne sait pas. Pas d'invention de sources ou de chiffres — il signale l'incertitude.

### Votre partenaire, pas votre miroir
Il challenge vos idées si elles semblent fausses. Un bon outil pousse, il ne valide pas tout.

## [thanks] Maîtrisez Claude *vraiment*
Tous les use cases, tous les formats, toutes les industries.
- Suivez NAIOM pour un prompt utile chaque semaine
- Save · Share · Commentez votre use case préféré
```
