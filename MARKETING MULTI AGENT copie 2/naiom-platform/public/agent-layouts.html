Ce qu'on mesure dans chaque dashboard agent

Peu importe le layout choisi, chaque agent aura ces métriques suivies. Aperçu de ce qui va être tracké et affiché.

Consommation
  • — Tokens input / output (live)
  • — Coût USD par jour / 7j / 30j
  • — Budget mensuel vs dépensé (barre)
  • — Cache hits & économie
  • — Appels tool (web_search, web_fetch)
Performance
  • — Latence moyenne réponse
  • — Taux de succès (vs erreurs)
  • — Uptime (incidents / 30j)
  • — Nb conversations actives
  • — Nb messages total
Production
  • — Livrables produits (count)
  • — Top formats (deck, post, email…)
  • — Dernière activité (timestamp)
  • — Livrables validés vs drafts
  • — Historique conversations 7j

Où on lit les métriques ?

Tokens + coût : API Anthropic Admin (usage endpoints) ou tracking local via le SDK. Latence + succès : telemetry ajoutée dans /api/chat. Livrables : scan du dossier agent. Conversations : localStorage (aujourd'hui) ou base serverless (à ajouter). Je peux tout brancher en vrai une fois que tu valides un layout.
